1. Create Design
- Design an agent that will run on multiple nodes in a cluster. The agent exchanges in-memory data with peer agents using NVidia NIXL framework, to measure memory transfer performance between nodes. 
  Split the design into the following sections - 
  1. Agent Design: a.bootstrap b. inter-agent communication via NIXL c. memory performance test suite d. Persisting performance data 
  2. Controller: a. startup and agent rendezvous using NIXL b. manage performance tests (kickoff, completion) c. Collect and collate perf data
  3. Topology discovery: a. Graph construction b. [Graph vizualization]
  4. Test Harness: a. Setup simulated test environment

  There will be separate modules to implement each design section.
  How agents bootstrap
  1) Redezvous: Controller starts at well known IP/port, registers a mem buffer with NIXL
    a) Controller is started with number of agents for given performance test
    b) Allocs buffer with size that scales with num agents (Each agent gets a 'slot' in this buffer)
    c) Writes number of agents at offset 0 in buffer (this will be used by agents to learn number of peers)
    d) Serializes its registered buffer & its NIXL endpoint metadata into blobs.
    e) Listens for agent connections
    f) When agent connects, Controller sends it the blobs and also its agent ID
    g) Agent uses this ID to decide the offset in the controller buffer in which to write 
    h) Each agent writes its NIXL endpoint metadata + buffer blob into its slot in the buffer
    Once every agent connects and completes this step, the Controller buffer contains all the information needed for any agent to talk with another peer
